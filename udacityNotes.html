Lesson 2

Style - CSS

AVOID REPETITIONS, USE CSS.

Repetitions can cause multiplication of errors (we repeat an error every time we copy/paste the fragment of code.)
Avoiding repetitions makes our code more clear and legible to us and other developers. It also saves our time (to copy/paste:) and space in our code.

Initial html tags: !DOCTYPE html, html and link to a style file (in HEAD)! (In scratchpad.io you can style code in between an opening and closing <style> tag.)

In CSS we define a style for an element. The child element inherits after parents' elements --> Cascade.

Use Selectors to give a style for the element you want. We can make a style for all the tags (i.e. all paragraphes) or for a class.

The Box Model:
  • margin
  • border
  • padding
  • content

Box-sizing should fit into every browser, so we need to use the browser specific prefixes - webkit, moz, ms.
Sizes of boxes can be set in pixels or percentages.

Divs are boxes. To improve a layout we use flexible boxes, more about them in a guide to flexbox. To deal with sizing issues we can also set box-sizing: border-box;
